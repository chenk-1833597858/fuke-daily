## 核心思路

### 核心原则：替身写作法

**写故事不是"设计剧情"，是"把自己放进去，看会发生什么"。**

**必须用第一人称「我」来写。** 不是「她」不是「主角」，是「我」。

### 核心规则：分数定结构，不是定结果

分数是写之前的工具，不是写之后的评价。

写之前先做这件事：
1. **定目标分数**（用户要求50分还是40分）
2. **反推需要几场什么级别的冲突** — 50分需要至少：一场9分 + 两场7-8分 + 三场6分 + 若干过渡
3. **如果当前题材撑不起这个分数** → 加多战场/加多对立面，合理扩写
4. **分数不能虚标** — 8分必须有人被打/被毁/见红/关系彻底断裂。6分必须有一方明确被伤害。4分必须有人当面不让。逐级对表：

| 分 | 问题（从0问到10，是就停） | 必要条件 | 真实因果链实例 |
|----|--------------------------|---------|--------------|
| 0 | 是不是日常？ | 举手之劳，顺手而为 | 陌生人扶门、同事倒水、邻居收快递 |
| 1 | 是不是应付？ | 心里不情愿但表面应付过去 | 亲戚问工资说「还行」、说「快了其实没动」 |
| 2 | 是不是顶撞？ | 口头顶撞，直接反驳对方 | 会议上直接说领导方案不行 |
| 3 | 是不是吵架？ | 激烈争吵，情绪升级 | 情侣因为前任联系大吵一架 |
| 4 | 是不是拉黑？ | 删好友/拉黑/断联 | 被分手后删掉所有联系方式 |
| 5 | 是不是决裂？ | 彻底决裂，老死不相往来 | 十年闺蜜因为借钱反目成仇 |
| 6 | 是不是打架？ | 动手打架，肢体冲突 | 占车位不让还骂人，动手了 |
| 7 | 是不是报复？ | 蓄意报复，损人利己 | 被出轨后把聊天记录发到家族群 |
| 8 | 是不是害命？ | 危及性命，谋害他人 | 酒驾撞人后逃逸 |
| 9 | 是不是灭门？ | 灭门之祸，牵连全家 | 高利贷追债放火烧了全家 |
| 10 | 是不是连坐？ | 一人犯罪全家遭殃，甚至牵连亲族 | 拐卖儿童团伙头目落网 |

**用法**：拿一条因果链，从0开始问，问到第一个「是」就停——那就是这条链的分数。不需要精确计算，只需要强制判断这到底属于哪一级。如果两个级别都"有点是"，往上取（因为冲突烈度宁高勿低）。

**搜索因果链**：因果推断库已导入 Qdrant 向量数据库（`http://127.0.0.1:6333`，集合 `causal_chains`），支持语义搜索。搜一个情境就能找到所有语义相近的因果链，比自己翻 JSON 快得多。用法详见 `references/causal-library-qdrant.md`。

**⚠️ 评分规则——只列因果链，不给分**：
- 用户明确指出「你的评分假的很」「不要乱改评分」——搜到因果链后只需列出 果、因[]、缘[]，由用户自行判断烈度。
- 入库前必须给用户看一遍（「入库前要给我看了以后才能入库」）。

5. **写不出高烈度冲突就让DeepSeek写** — 安全限制是模型的问题，不是故事的问题。关键冲突场景（7分以上）如果感觉写不动，直接交给DeepSeek子agent起草。

### 写之前的因果定调（必须先定义人物，再推因果）

**⚠️ 铁律：不允许先射箭再画靶。必须先定义人物的一切，再从人物现状出发推导因果。**

动笔之前必须完成以下四步，放在骨架最前面：

#### 第一步：定义女主的一切（工作、生活、人际关系）

**女主的工作：**
- 当前工作是什么？（有/没有？什么类型？）
- 以前做过什么？（技能、学历、梦想）
- 经济独立吗？（这是她能离开的关键）

**女主的生活：**
- 日常是什么样的？（具体、细节、不是笼统的"做饭打扫"）
- 社交圈有多大？（有闺蜜吗？有朋友吗？有支持系统吗？）
- 爱好是什么？（以前有什么，现在还有吗？）
- 状态如何？（麻木？快乐？焦虑？具体表现是什么？）

**女主的人际关系：**
- 丈夫：什么职业？什么性格？核心盲点是什么？
- 闺蜜：存在吗？敢说真话吗？
- 父母：在不在身边？什么态度？
- 朋友：有自己的社交圈吗？

#### 第二步：定义读者

**目标读者：**
- 年龄、性别、状态

**读者的痛点：**
- 她们在关系里最普遍的经历是什么？
- 她们放不下的是什么？

**读者想看的结果：**
- 她们想要但没得到的结局是什么？

**读者会产生的情绪：**
- 前期：？
- 中期：？
- 后期：？

**这个经历是大多数还是一小部分？**
- 如果只有一小部分有 → 换题材或删掉重来
- 如果大多数有 → 继续

#### 第三步：因果推倒（基于人物定义，不预设剧情）

从人物现状出发，逐级追问：

**因果推倒1：她为什么容忍了这么久？**
- 因（她的现状）：基于第一步定义
- 缘（触发条件）：社会文化、原生家庭、没有退路
- 果（她的状态）：继续忍耐 / 终于爆发

**因果推倒2：她为什么终于爆发了？**
- 因（她的内在变化）：基于第一步定义
- 缘（触发事件）：具体事件，基于人物性格
- 果（她的决定）：提出离婚 / 净身出户

**因果推倒3：他为什么后悔了？**
- 因（他的内在）：基于第一步定义
- 缘（触发事件）：具体事件，基于人物性格
- 果（他的状态）：慌了 / 挽回 / 后悔

**因果推倒4：她为什么拒绝了他？**
- 因（她的内在）：基于第一步定义
- 缘（触发事件）：具体事件，基于人物发展
- 果（她的状态）：放下 / 独立 / 过得很好

#### 第四步：验证因果链（检查是否先射箭再画靶）

每个因果链必须回答：
- [ ] 这个因是基于人物定义的吗？
- [ ] 这个缘是基于人物性格的吗？
- [ ] 这个果是基于人物发展的吗？
- [ ] 有没有预设剧情？（如"开花店""写作"等）
- [ ] 如果删掉这个场景，人物还成立吗？

**反面示例（❌ 先射箭再画靶）：**
- ❌ 女主是文学系研究生 → 她发现自己不会写作了 → 她觉醒了
- ❌ 女主没有工作 → 她开了间花店 → 她独立了
- ❌ 女主喜欢写作 → 她重新拾起了笔 → 她成功了

**正面示例（✅ 基于人物定义）：**
- ✅ 女主是全职主妇，没有经济独立 → 她发现自己连买杯咖啡都要问他要钱 → 她意识到自己被困住了
- ✅ 女主没有社交圈 → 她发现自己连一个可以说心里话的人都没有 → 她意识到自己孤立无援
- ✅ 女主有照顾人的技能 → 她发现自己把所有人都照顾好了，唯独没照顾好自己 → 她决定先爱自己

#### 因果定调格式

```
## 因果定调

### 人物定义
- 女主：林知意，28岁，全职主妇，文学系研究生未毕业
- 男主：沈砚之，32岁，科技公司创始人，理性自负
- 闺蜜：偶尔联系，但不太敢说真话
- 父母：不在身边

### 读者定义
- 目标读者：25-35岁女性
- 痛点：付出不被看见，想离开但不敢
- 想看的结果：她离开不是报复，是自救
- 情绪曲线：心疼→解气→释然
- 普遍性：大多数（70%以上有过类似经历）

### 因果推倒

因果推倒1：她为什么容忍了这么久？
因：她没有工作，没有经济独立（基于人物定义）
缘：社会文化告诉她"女人要忍耐"（基于社会环境）
果：她继续忍耐，但越来越麻木（基于人物状态）

因果推倒2：她为什么终于爆发了？
因：她发现自己已经不认识自己了（基于人物状态）
缘：他说了那句"除了我谁要你"（基于人物性格）
果：她终于爆发了（基于人物发展）

...（以此类推）

### 冲突级别需求
[目标分数]→[需要的场景配置]
例：50分 → 需要至少1场9分 + 2场7-8分 + 3场6分 + 若干过渡
```

写完之后检查：每个场景的分数是否真实对应了冲突级别表里的必要条件。如果没有，要么是分虚高了（那就降分），要么是冲突没写够（那就扩写）。

### 核心原则#2：活人感——冲突是人物活着的证据

冲突不是剧情工具。读者记不住一个"性格描写"里的人物——只有当那个人跟什么撞上了、让他疼了、不得不做出选择，他才从纸上站起来。

每场冲突必须做的事：让角色做一个逃不掉的选择。
- 两个好东西只能保一个（守店还是保女儿？）
- 两个坏东西必须吞一个（认输走人，还是以命换命？）
- 不说话也是一种选择（沉默的代价）

检测方法：问自己——这场冲突如果去掉，读者会不会觉得角色是纸片人？如果去掉也不影响，这场就没生出力。

> **如果是我——以我的性格、我的经历、我放不下的那些事——被放到那个世界里，遇到了那些人、那些条件，我会遇到什么？**

把"我"放进去，因（性格/创伤/本能）和缘（环境/搭档/条件）自然会引出果。

### 与因果分析技能的关系

|  | 因果分析技能（已有） | 这个技能 |
|--|-----------|---------|
| 起点 | 一个已有的结果 | 主角的因 + 初始条件（缘） |
| 方法 | 从果倒推因缘链 | 从因+缘顺推果 |
| 目的 | 理解为什么发生 | 发现会发生什么 |
| 输出 | 分析报告 | 场景清单 |

可以组合使用：先用因果分析拆解同类故事的因缘结构，再回到这个技能把自己放进去顺推。

## 核心原则（必须放在一切之上）

### 冲突的本质：每一次都要做选择

**冲突不是两个人吵架。冲突是把角色逼到「两个好东西选一个」或者「两个坏东西必须挑一个」的位置。**

检测方法：写出你的场景，问自己——这个场景里，角色做了什么选择？
- 选了A放弃了B → ✅ 有冲突
- 什么也没选，只是"心情不好"或"表达不满" → ❌ 这不是冲突，是情绪描写

好的冲突场景，角色走出那个场景时已经**不是走进去时的那个人了**。

### 活人感的来源：情绪写的是读者，不是角色

**小说写的不是主角，写的是读者本人。**

要让读者带入，不能靠描写角色的情绪——要靠**勾起读者自己的经历**。

#### 动笔前的第一问：读者是谁？

本技能默认读者画像：**20-45岁女性**。

**倒推读者的因（调用因果分析技能）：**
- 打开因果分析技能，目标设置为「20-45岁女性读者在亲密关系中最高频的共同经历」
- 倒推：她们放不下什么？什么场景让她们一看到就想起自己？
- 输出结果作为主角的因的原料

已验证的高频因：
- **不被看见** — 说了很多次，对方一次没记住
- **付出被当作理所当然** — 照顾一切，没人问过你累不累
- **退让了但没有尽头** — 你退一步，对方进一步，最后没你的位置了
- **被决定** — 重要的事没人问过你的意见
- **等了很久但等不到** — 不是对方不给，是对方根本不知道你在等

#### 选材铁律

不要写读者没经历过的（如「下跪」「以命相搏」「守面馆」）。
**要写读者经历过的因，但给读者一个她没有的果。**
- 读者经历过的因：「不被看见」
- 读者想要的果：「被看见，而且那个人付出了代价」
- → 这就是追妻火葬场的核心

#### 冲突选材的因果路径

```
[读者的因] → [角色的因] → [角色的果]

读者自己的经历    角色替读者活一次    角色替读者得到一个读者没得到的结局
```

### 冲突写的是角色，情绪写的是读者

冲突让角色疼——那只是骨架。
情绪让读者跟着疼——那才是血肉。

让读者跟着疼的方法不是描写角色的疼，是**留出缝隙让读者把自己的经历填进去**：
- 「他忘记了」——读者想起自己那一次
- 「她等了很久」——读者想起自己在等什么
- 「他没站起来」——读者想起那个没站起来的人

### 冲突烈度分配原则：前重后轻

**前期高烈度，后期低烈度。**

读者买单的是前期——第一页就要让她疼。「这说的就是我」这个感觉必须在第一个场景就建立。

**检测方法：** 把场景清单按烈度排序。如果中间或结尾的冲突比开头强，说明结构反了。

#### 整体情绪曲线：读者需求满足曲线

完整的情绪起伏不是「一个比一个狠」，而是：

```
前期（前2-3场）→ 需求不被满足，烈度较高。读者看到「这说的就是我」，但角色还没得到她想要的。
中期（中间3-4场）→ 需求逐渐被看见，烈度开始降低。角色开始行动/离开/反击，但还没有彻底解决。
后期（最后1-2场）→ 需求被满足。烈度最低，但情绪最完整——角色走出来了，读者也跟着走出来。
```

推荐分布模式：
- **7-8个场景**：中-高-中-高-中-低-低
- **5-6个场景**：中-中-高-中-低

**验证标准：**
1. 第一场是否有7分级别的情感杀伤力？
2. 前半段（前4场）是否承担了全篇最高烈度？
3. 后半段（后4场）烈度是否逐渐降低？
4. 尾声是否是最低烈度但最高完整度？
5. 读者需求是否在前期不被满足、后期被满足？

写法上的落地：
- **前期**：选最痛的那个具体瞬间直接开场。越具体越好。
- **前期开局必须是价值观/信念冲突**：不要为具体事情吵。要为**两个人对同一件事的理解完全不同**而吵。一个人道歉也解决不了。**不吵架，有个屁的冲突。** 如果开局连架都吵不起来，事件冲突就根本不存在——那不是解决的方法不对，是冲突还没开始。
- **后期**：烈度降低不是剧情变弱，是情绪从「疼」变成「愈合」。

#### 配角原则：没有多余的子弹

电影里没有一颗子弹是多余的。短篇小说也是。配角要么不出现，出现了就必须有明确的用处：

1. **帮主角** — 盟友，护着主角一起去干
2. **冲突对象** — 跟主角面对面互不相让
3. **帮冲突对象的** — 站在对面，给主角增加难度
4. **推动剧情的** — 一句话/一个信息让冲突升级或转向

不在这四条里的配角，删掉。一句带过都算多。

### 写长：多战场推进，不是多场景堆砌

写长不是把7个场景硬撑成15个——是**把同一场冲突放到不同的战场上打，每一场都是真打，没有过渡场景**。

核心原则：**每一个场景必须有一个人跟另一个人面对面互不相让。** 如果写不出这个场景的对抗方和交锋链，这个场景就不存在。

真正有价值的扩展方式：

**1. 多战场：同一场仗换地方打**

不是同一件事吵两遍——是同一个价值观冲突，在不同场景下重新爆发。

举例：
- 战场一：家里餐桌——冰水，价值观正面撞（8分）
- 战场二：电话——他说「你怎么不跟我说」，她说「说过很多次了，你没听过」（4分）
- 战场三：咖啡店——她说「上次你也是这么说的」（5分）
- 战场四：她家楼下——他说「你连吵都不吵就判死刑，公平吗」，她说「去年三月四十分钟，你戴着耳机没摘」（7分）

每一个战场都是真打，只是换了一个地方、换了一个新的切入点。**不是同一个冲突写四遍，是同一个伤口被从四个角度戳。**

**2. 多对立面：主角同时跟多个角色干架**

如果只有一个冲突对象，再多场景也是同一条线。加入第二个冲突对象（妈妈、老板、闺蜜反过来站在对立面），让主角同时在多条战线上打。

但注意：**每一条战线都必须是真实的对抗。** 如果妈妈只是打电话劝一次就完了，那不是冲突线，是注水。妈妈必须跟主角有真正不可调和的分歧——比如妈妈觉得「女人就是要忍，不然你还能怎样」，主角觉得「我为什么要忍」——这是两代人的价值观碰撞，值得单独展开。

### 交付前自查

- [ ] 这篇小说的读者是谁？（默认20-45岁女性）
- [ ] 他们的「因」是什么？
- [ ] 角色的因和读者的因重合了吗？
- [ ] **全文使用第一人称「我」**——不是「她」不是「主角」，是「我」
- [ ] 前三个场景里哪个冲突强度最高？是不是最高峰放在开头？
- [ ] 烈度是从前到后逐渐降低的吗？
- [ ] 第一场是价值观冲突，不是事实冲突？
- [ ] 不在这四条里的配角删了吗？帮主角/冲突对象/帮冲突对象的/推动剧情

### 交付物格式：骨架优先

**重要：用户要的是骨架（skeleton），不是正文（full prose）。**

当用户要求你出剧情/冲突/构思时，交付物必须是以下格式的**骨架**，而不是写好的小说正文：

**骨架必须包含的七个要素：**

1. **核心设定** — 一句话说清「我」是谁、「他」是谁、冲突本质是什么
2. **冲突曲线总览** — 表格：场次、强度、对阵、战场、类型，附总分和硬门槛自查
3. **逐场交锋链骨架** — 每场只写攻防逻辑，不写场景描写/对话铺陈：
   - 触发条件
   - 攻防链（攻1→防1→攻2→防2…至少2-4轮）
   - 决胜/败退方判定
4. **配角体系** — 四线角色表（帮主角/冲突对象/帮冲突对象的/推动剧情），每人附关键动作
5. **意象系统** — 初现→callback 的对应关系链
6. **待验证项** — 用户确认后才能定下的设计决策

**反面示例（避免❗）：**
- ❌ 写了完整对话和场景描写——这是正文，不是骨架
- ❌ 只列了场景标题没有攻防链——缺冲突证据
- ❌ 没有评分和曲线检查——缺结构验证

**正面示例（已通过骨架中的案例结构）：**
- 用户通过案例库 `references/passed-cases.md` 中的「我不干了」「他用了三年」骨架格式

> 每条攻防链写到「他做了什么→我回了什么→他再反击→我再回」层级即可，不需要铺成场景描写。每场控制在3-5行要点，7场骨架一般在600-1000字。

### 追妻火葬场特化指南

> 详细参考：`references/trace-fire-pursuit.md`

追妻火葬场是短篇中最常见也最吃技巧的类型。以下规则基于用户反复验证通过的经验。

**1. 读者的因（必须倒推）**
- 说了很多次，对方一次没记住
- 付出被当作理所当然
- 退让了但没有尽头
- 等了很久等不到
- 最后不是爆发，是有一天突然不疼了

**2. 角色结构（必须三条线）**
- 第一线：我 vs 他（主冲突线，价值观碰撞）
- 第二线：闺蜜/朋友（贯穿全篇的清醒敲打者——不说「你做得对」也不说「你快回去」，而是逼她面对「你到底还爱不爱他」这个问题）
- 第三线：长辈/婆婆（独立见证者——三段式贯穿：①第一段建立立场和可信度 ②第二段提供关键信息 ③第三段收束主题）

**3. 和好结局铁律**
- 不能只是「他说他改了」——必须有不可逆的代价展示（股权协议/辞职签名/某个明确不能撤销的行动）
- 最好有一个标志性意象的callback（如「常温的水」「饺子馅」）

**4. DeepSeek评分阈值**
- 追妻火葬场目标：≥50分（普通短篇≥20分）
- 不过必须逐场修改后重新评分，不能越过问题直接通过
